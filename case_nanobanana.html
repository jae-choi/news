단일 LLM 디자인 에셋 파이프라인

Unified LLM Design Asset Pipeline

Customer Problem Solving Case 5

“다중 비전 스택을 단일 VLM과 OpenCV로 줄이다”

“Collapsing a multi-vision stack into a single VLM plus OpenCV”

YOLO, SAM 같은 다중 비전 모델 조합은 정확하지만 운영 비용과 인프라 종속성이 컸습니다. 이 사례는 단일 VLM과 OpenCV 보정 기법을 결합해 운영 복잡도를 줄인 디자인 에셋 생성 파이프라인을 설명합니다.

Combinations of vision models such as YOLO and SAM can be accurate, but they introduce high operating cost and infrastructure dependence. This case explains a design asset pipeline that reduced operational complexity by combining a single VLM with OpenCV-based correction techniques.

1. Need: 도입 배경

1. Need: Background

디자인 에셋 생성 과정에서 객체 탐지, 누끼, 확장 같은 단계를 여러 모델로 나누어 운영하면 개발과 배포 비용이 빠르게 증가했습니다.

When design-asset generation required separate models for detection, segmentation, and extension, development and deployment cost grew quickly.

2. Problem Definition: 문제 정의

2. Problem Definition: Problem Statement

다중 딥러닝 모델 구조는 GPU 비용과 운영 복잡도가 크고, 특정 모델 장애가 전체 파이프라인 실패로 이어질 위험이 있었습니다.

반면 단일 모델로 줄이면 표현력이나 후처리 품질이 낮아질 수 있다는 trade-off가 존재했습니다.

A multi-model deep-learning stack carried high GPU cost and operational complexity, and a failure in one model could break the entire pipeline.

On the other hand, reducing everything to a single model risked lower fidelity and weaker post-processing quality.

3. Key View: 접근 방식

3. Key View: Approach

핵심은 모든 품질 문제를 모델 수 증가로 해결하지 않고, VLM의 한계를 수용한 뒤 비전 후처리를 창의적으로 결합하는 것이었습니다.

특히 배경을 네온 그린으로 처리한 뒤 OpenCV로 제거하는 방식을 도입해 단일 모델의 한계를 보완했습니다.

The key was not to solve every quality issue by adding more models, but to accept the limitations of a VLM and creatively combine post-processing.

In particular, a neon-green background strategy followed by OpenCV removal compensated for weaknesses of the single-model approach.

4. Design / Framing: 아키텍처 설계

4. Design / Framing: Architecture Design

입력 이미지를 VLM에 전달해 객체 위치와 배경 분리 방향을 생성한 뒤, 후처리 단계에서 프롬프트 + OpenCV 조합으로 실제 누끼와 확장을 정제했습니다.

결과적으로 GPU 집약적 다중 스택 대신, 단일 모델과 가벼운 이미지 처리 체인으로 운영 패턴을 단순화했습니다.

Input images were passed into a VLM to generate object localization and background separation guidance, then a prompt + OpenCV combination refined cutout and extension in post-processing.

The result was a simpler operating pattern built on a single model plus lightweight image processing instead of a GPU-heavy multi-stack.
