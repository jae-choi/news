Proof of Concept (POC)

단일 LLM 기반 디자인 에셋 생성 파이프라인

비싼 전용 비전(Vision) 딥러닝 모델들을 걷어내고, 오직 Gemini Vision 모델의 프롬프트 엔지니어링과 OpenCV 크로마키 기법만으로 배경 제거(누끼)부터 이미지 확장(Outpainting)까지 일괄 자동화한 파이프라인입니다.

웹툰, 커머스 등 마케팅 에셋을 대량 생산하는 부서에서는 캐릭터 썸네일 추출(누끼 따기), 글자 지우기, 레이아웃 16:9 확장과 같은 반복적인 디자인 작업에 엄청난 비용과 인력을 소모합니다.

이 문서(POC)는 여러 개의 전용 딥러닝 모델(YOLO, SAM, SD 등)을 운영하는 무거운 인프라 구조를 버리고, 단 하나의 거대 언어 모델(Gemini Vision)과 Nano Banana 크로마키 기법을 통해 통합 디자인 자동화 파이프라인을 구축한 기술적 성과를 다룹니다.

1. Problem

복잡한 비전 파이프라인과 비싼 GPU 인프라 종속성

다중 딥러닝 모델 유지보수의 늪

일반적으로 이미지 속 인물을 찾고(YOLO), 배경을 지우며(SAM), 빈 공간을 채워넣으려면(Stable Diffusion) 3~4개의 각기 다른 고비용 AI 모델들을 각각의 GPU 서버에 배포하고 연결해야 합니다. 이는 관리 포인트를 기하급수적으로 늘립니다.

API 기반 투명 배경 생성의 기술적 한계

클라우드 서비스(Gemini 등)의 범용 비전 모델은 이미지 생성은 뛰어나지만, 알파 채널(투명 배경, PNG)을 깔끔하게 반환하는 누끼 따기 능력은 구조적으로 불가능에 가까웠습니다. 모델이 체커보드 패턴을 그리는 등 오작동이 빈번했습니다.

2. Solution

Gemini 3.1 VLM 모델 기반 올인원(All-in-one) 비전 프롬프팅

여러 개의 비전 특화 모델들을 모두 폐기하고, 오직 단일 거대 시각 언어 모델(Gemini 3.1 Pro/Flash)만 사용하여 객체 탐지, 이미지 확장(Outpainting), 글자 지우기(Inpainting)를 모두 처리하는 백엔드 서버(FastAPI)를 구축했습니다.

특히 불가능에 가깝던 투명 배경 처리(누끼) 문제는 '모델에게 배경을 형광 녹색으로 칠하게 한 뒤 OpenCV로 뚫어내는 (Nano Banana 기법)' 창의적 엔지니어링으로 완벽하게 해결했습니다.

4. Technical Decisions

비용 한계 극복을 위한 창의적인 엔지니어링 의사결정

SAM 모델 등 전용 인프라 운용 vs Nano Banana 크로마키 기법

완벽한 누끼(배경 제거)를 위해 SAM(Segment Anything Model) 서버를 구축하면 GPU 비용이 크게 상승합니다. 반면, API 기반 VLM 모델은 투명 배경(Alpha)을 제대로 렌더링하지 못하는 구조적 맹점이 있었습니다.

해결: Prompting과 OpenCV의 결합 (Nano Banana) 모델에게 투명 처리를 기대하는 대신, '배경을 완벽한 네온 그린(#00FF00)으로 덮어라'는 강력한 프롬프트를 주입했습니다. 이렇게 렌더링된 이미지를 FastAPI 백엔드에서 OpenCV 라이브러리를 통해 전통적인 크로마키 기법으로 초록색만 뚫어버리는 우회(Workaround) 전술을 사용하여 GPU 비용을 완전히 소거했습니다.

절대 픽셀값 예측 vs 좌표 정규화 기반 공간 추론

다양한 해상도(Resolution)의 이미지가 들어올 때, 모델에게 특정 객체의 Bounding Box를 정확한 절대 픽셀 좌표(예: 1024x768)로 찾으라고 명령하면 인식률이 크게 요동칩니다.

해결: 0~1000 스케일 맵핑 및 JSON 응답 강제 원본 해상도에 상관없이 모델에게 이미지를 가로세로 0부터 1000까지의 상대적 비율로 인식하게 한 뒤, [ymin, xmin, ymax, xmax] 형태의 정규화된 JSON 데이터로만 출력하도록 시스템 프롬프트를 고도화했습니다. 백엔드에서는 이를 다시 원본 해상도로 곱하여(Scale-up) 완벽한 크롭(Crop) 안정성을 확보했습니다.

5. Business Impact

고비용 비전 인프라 제로화

마케팅 썸네일 생성을 위한 무거운 딥러닝 서버군(YOLO, SAM, SD)을 경량 백엔드 API 서버 1대로 100% 통합 및 교체했습니다. 이로 인해 수천 달러에 달하던 월간 GPU 유지보수 비용을 Gemini 토큰 비용 수준으로 파격적으로 줄였습니다.

통합 에셋 파이프라인의 완성

  • 크리에이티브 워크플로 통합: 디자이너가 여러 툴을 번갈아 쓸 필요 없이, 단일 API 플랫폼에서 '인물 누끼 추출 → 텍스트 삭제 → 여백 16:9 확장'까지 한 번에 자동화하여 에셋 생산성을 극대화했습니다.
  • 비개발 모델 파인튜닝의 한계 극복: 모델 자체의 한계(투명도 렌더링 불가 등)를 외부 코드 레벨의 엔지니어링(OpenCV 크로마키)으로 우회 결합하여, AI의 결함을 유연하게 보완하는 아키텍처 방법론을 증명했습니다.
