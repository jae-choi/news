Proof of Concept (POC)

의도 분석 기반 멀티 에이전트 사내 검색 아키텍처

엔터프라이즈 환경에서 직원들은 매일 수천 건의 사내 문서를 검색하며 의사결정을 내립니다.

이 문서(POC)는 방대한 데이터 크기와 다양한 질의 패턴을 가진 사내 지식 관리 시스템(KMS)에서 의도 분류 모델(Intent Model)과 멀티 에이전트 라우팅을 통해 전문가 수준의 검색 아키텍처를 어떻게 구축했는지 설명합니다.

1. Problem

복잡한 사내 데이터 환경에서의 검색 정확도 저하와 의도 단절

복합 질의(Complex Query) 처리 불가

사내 사용자는 '최신 프로젝트의 개발 아키텍처와 마케팅 일정을 요약해줘'와 같이 기획, 기술, 인사 등 여러 도메인의 의도가 섞인 질문을 던집니다. 기존 단일 검색기(Vector Search)는 이 문맥을 분리하지 못하고 무작위로 섞인 문서를 가져옵니다.

단일 LLM 프롬프트의 한계와 환각

모든 사내 규정과 기술 문서를 하나의 프롬프트에 구겨 넣고 거대 모델 하나로 답변을 생성하려 하면, 토큰 비용이 낭비될 뿐만 아니라 관련 없는 부서의 문서가 검색되어 '환각(Hallucination)' 현상이 빈번하게 발생합니다.

2. Solution

의도 분류 모델(Intent Model)과 멀티 에이전트 라우팅의 결합

사용자의 질의를 1차적으로 '의도 분류 모델과 Gemini의 하이브리드 계층'에 통과시켜 핵심 의도(Intent Labels)를 분리 및 추출합니다.

이후, 추출된 의도에 맞는 특화된 분야별 전문 에이전트(인사, 운영, 개발 등)를 동적으로 호출합니다. 각 에이전트는 자신만의 좁혀진 권한으로 Vertex AI Search의 데이터 스토어(Datastore)를 쿼리하여 가장 신뢰할 수 있는 사내 정보를 획득합니다.

4. Technical Decisions

정확도 향상과 인프라 최적화를 위해 거쳤던 치열한 엔지니어링 고민

의도 분석 계층 분리 vs 단일 거대 모델 사용

사용자의 쿼리를 받을 때마다 무거운 프롬프트를 싣어 거대 모델(LLM) 하나가 모든 의도를 판단하게 하면, 시스템 응답 속도(Latency)가 저하되고 API 호출 비용이 급격히 늘어납니다.

해결: 경량 모델(Classifier) 선행 배치 문맥 전체를 읽지 않고 빠르고 정확하게 '의도 라벨(Intent Labels)'만 분리하는 별도의 하이브리드 분류 계층을 앞단에 배치했습니다. 이 라우터가 전문 에이전트를 먼저 호출하게 함으로써, 불필요한 토큰 낭비를 막고 검색 파이프라인의 속도를 최적화했습니다.

순수 Vector DB 구축 vs Vertex AI Search(Agent Builder) 편입

오픈소스 Vector DB를 직접 컨테이너에 띄워 데이터를 색인하면 제어권은 완벽하지만, 엔터프라이즈의 보안(Security) 인증 및 데이터 크기(Scale)에 비례하는 끔찍한 인프라 관리 부채가 발생합니다.

해결: Cloud Storage 연동의 관리형 Datastore 도입 정제된 KMS 데이터를 Cloud Storage 버킷에 올리고, 이를 Vertex AI Search의 Datastore로 다이렉트 인덱싱하는 구성을 채택했습니다. 이는 구글 인프라의 완전 관리형 하이브리드 검색 및 Reranker 기능을 백엔드 오버헤드 없이 즉시 사용할 수 있는 아키텍처적 결단입니다.

5. Business Impact

사내 정보 탐색의 비효율 제거

의도 분석 모델과 멀티 에이전트의 연계로, 복잡한 질문에서도 마치 부서별 전문가가 대답하는 것과 같은 정확도 높은 답변이 실시간으로 반환되어 엔터프라이즈 생산성을 대폭 강화합니다.

모듈형 데이터 파이프라인 정착

  • 플러그앤플레이(Plug-and-Play) 인덱싱: Cloud Storage에서 스테이징된 데이터를 Vertex AI Search로 즉각 매핑하는 파이프라인을 구축하여, 향후 타 부서나 완전히 다른 형태의 신규 데이터베이스라도 쉽게 편입시킬 수 있는 확장성을 획득했습니다.
  • Cloud Run 기반 배포 유연성: 모델 엔드포인트와 애플리케이션 프론트엔드를 Cloud Run 기반 컨테이너로 각각 독립 배포하여 성능 추적, 부하 분산 및 보안 인프라 구성을 완벽히 분리했습니다.
